# Chapter 4 — Build the Page + Checkout in 3 Hours

This chapter is about one thing: getting to a live URL with a working payment link.

## The 3-Hour Build Sprint

### Hour 1: Message and structure
Build these sections only:
1. Hero (outcome + timeframe)
2. Problem (why they are stuck)
3. Offer stack (what’s included)
4. Pricing (Lite/Pro/Sprint)
5. FAQ
6. CTA

If a section doesn’t increase checkout intent, remove it.

### Hour 2: Checkout wiring
- Create Stripe products for each tier
- Generate checkout links
- Put real links on each CTA button
- Test checkout in live mode with small real transaction if possible

### Hour 3: Delivery path
- Create success page
- Add access instructions on success page
- Send confirmation email copy (manual or automated)
- Add support contact

## Sales Copy Rules

- Headline must state outcome + speed
- Bullets must answer: “What do I get right now?”
- Pricing must be visible without scrolling too far
- CTA must be action-based (“Get Instant Access”)

## Required Conversion Elements

- Social proof placeholder (or founder note if no testimonials yet)
- Urgency line (launch pricing ends at [time])
- Low-risk statement (clear support/refund policy)

## Stripe Setup Checklist

- [ ] Product names match landing tiers
- [ ] Checkout descriptions are clear
- [ ] Success URL routes correctly
- [ ] Customer email captured
- [ ] Confirmation email includes delivery links

## Chapter 4 Action Checklist

- [ ] Live page published
- [ ] Checkout links functional
- [ ] Success page ready
- [ ] First test transaction validated

No page + no checkout = no sprint. Ship this before writing more content.
